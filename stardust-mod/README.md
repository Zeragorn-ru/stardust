# Stardust Mod

Один NeoForge-мод для клиента и сервера (Minecraft 1.21.1).

## Зачем он нужен

Первая версия — каркас общей runtime-интеграции:

- один jar кладётся и в клиентскую сборку, и на сервер;
- клиентская часть позже сможет получать настройки лаунчера/сервера;
- серверная часть интегрируется с плагином/модом **TAB** (NEZNAMY/TAB):
  выставляет игрокам бейдж (tab-префикс) и цветной ник в табе.

Важно: сам запуск клиента с `username`, `uuid` и `accessToken` сейчас делает
лаунчер. Для реального отображения серверных скинов в online-mode следующим
шагом нужен Yggdrasil/sessionserver слой на `auth-server` и подключение
`authlib-injector` при запуске клиента и сервера. Мод будет дополнять эту схему,
но не заменяет auth/session API Mojang.

## Интеграция с TAB

TAB устанавливается на сервер как отдельный мод. Stardust обращается к нему
только через `compileOnly`-зависимость `com.github.NEZNAMY:TAB-API` (JitPack);
TAB **не** включается в наш jar. Если TAB на сервере нет — интеграция молча
не активируется, мод продолжает работать.

Бейджи и цвета ников сейчас берутся из локального файла
`config/stardust-badges.properties` (создаётся пример при первом запуске).
Формат строки:

```
НИК = ПРЕФИКС/БЕЙДЖ | ЦВЕТ_НИКА
# например:
Steve = &b[Stardust] | &b
```

Файл перечитывается на `/tab reload`. Позже источником данных станет backend
Stardust.

## Сборка

Нужна Java 21+.

```sh
cd stardust-mod
./gradlew build
```

Готовый jar: `build/libs/stardust-mod-0.1.0.jar`.

На Windows без wrapper можно использовать установленный Gradle 8.10+:

```sh
gradle build
```
