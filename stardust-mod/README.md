# Stardust Mod

Один Fabric-мод для клиента и сервера.

## Зачем он нужен

Первая версия — каркас общей runtime-интеграции:

- один jar кладётся и в клиентскую сборку, и на сервер;
- клиентская часть позже сможет получать настройки лаунчера/сервера;
- серверная часть позже сможет интегрироваться с нашей авторизацией, скинами,
  whitelist/профилями и диагностикой.

Важно: сам запуск клиента с `username`, `uuid` и `accessToken` сейчас делает
лаунчер. Для реального отображения серверных скинов в online-mode следующим
шагом нужен Yggdrasil/sessionserver слой на `auth-server` и подключение
`authlib-injector` при запуске клиента и сервера. Мод будет дополнять эту схему,
но не заменяет auth/session API Mojang.

## Сборка

Нужна Java 17+.

```sh
cd stardust-mod
./gradlew build
```

На Windows без wrapper пока можно использовать установленный Gradle:

```sh
gradle build
```

После подключения wrapper добавим `gradlew` в репозиторий.
